# Humanizer（人性化改写）

一款 Clawdbot 技能：弱化文本中的「AI 写作痕迹」，使表达更自然、更像人写的。

## 安装

通过 ClawdHub 安装：

```bash
clawdhub install humanizer
```

## 用法

请智能体对文本做人性化处理，例如：

```
请对下面这段做人性化改写：[你的正文]
```

（英文环境也可使用：`Please humanize this text: [your text]`。）

或在编辑文档时直接调用。

## 概览

依据维基百科条目 [「AI 写作迹象」](https://en.wikipedia.org/wiki/Wikipedia:Signs_of_AI_writing)（由 WikiProject AI Cleanup 维护）。该指南基于对数千段 AI 生成文本的观察整理而成。

### 核心观点

> 「大语言模型用统计算法猜下一个词该写什么，结果往往趋向于在最大范围适用场景下、统计上最『保险』的那种写法。」

## 可识别的 24 类模式

### 内容类
1. **意义拔高** — 「标志着转折性时刻……」→ 改为具体事实  
2. **堆砌知名度** — 罗列来源却无上下文  
3. **浅层 -ing 分析** — 「象征着……体现着……」  
4. **宣传腔** — 「坐落于令人屏息的……」  
5. **模糊归因** — 「专家认为……」  
6. **套路化『挑战』** — 「尽管面临挑战……依然蓬勃发展」

### 语言类
7. **AI 高频词** — 「此外……见证……格局……」  
8. **回避系动词** — 用「作为……」代替「是」  
9. **否定式排比** — 「不仅是 X，更是 Y」  
10. **三项式** — 强行凑成三条  
11. **同义轮换** — 过度换词  
12. **虚假范围** — 在非可比维度上用「从 X 到 Y」

### 风格类
13. **破折号（—）滥用**  
14. **粗体滥用**  
15. **行内小标题列表**  
16. **标题式大小写（Title Case）**  
17. **表情符号装饰**  
18. **弯引号（“ ”）**

### 沟通类
19. **聊天机器人腔** — 「希望对你有帮助！」  
20. **知识截断免责声明** — 「尽管细节有限……」  
21. **谄媚语气** — 「问得好！」

### 废话与含糊
22. **赘语** — 「为了……」「由于……这一事实」  
23. **过度含糊** — 「或许可能大概」  
24. **万能积极收尾** — 「未来一片光明」

## 完整示例

**改写前（AI 腔）：**
> 此次软件更新彰显了公司对创新的承诺。而且，它带来无缝、直观且强大的用户体验——确保用户能高效达成目标。

**改写后（更自然）：**
> 软件更新加入了批处理、快捷键和离线模式。内测反馈大多积极，多数人表示完成任务更快了。
