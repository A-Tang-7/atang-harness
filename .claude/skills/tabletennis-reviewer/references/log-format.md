# tabletennis-reviewer 日志格式参考

写回时与用户指定的复盘日志保持一致风格。

## 文件结构

```markdown
---
tags: [乒乓, 成长记录, 2026]
created: YYYY-MM-DD
---

# YYYY 乒乓成长记录

> 统计截至 YYYY.MM.DD，累计 **N 次**打球，约 **X 小时**。

---

## N月

> [!summary] N月总结
> - 打球次数：**… 次**，时长 **…h**
> - …

### YYYY.MM.DD · Xh

**内容：** …

**收获**
- …

**不足**
- …

**改善计划**
- …（跟踪）

---

## 常见对手分析

| 对手 | 特点 | 应对策略 |
| … | … | … |
```

## 日条目字段

| 节 | 何时写 |
|----|--------|
| 内容 | 几乎总有：练球类型、对手、单人/多人 |
| 收获 | 有效变化、成功套路 |
| 不足 | 失误模式、意识漏洞 |
| 体会 | 心态、战术反思、长篇分析（可与不足并存） |
| 改善计划 | 下场可执行动作；需跟进加 `（跟踪）` |

节标题用加粗中文：` **收获** `，下列 `-` 列表。

## 跟踪标记

- 写法：行尾 `（跟踪）`，与现有日志一致
- 含义：跨场次持续跟进，下次 tabletennis-reviewer 复盘必问
- 典型来源：
  - 改善计划里自觉未养成的习惯（盯球、步伐、发球前思考等）
  - 待完成的总结/课程（如「总结正手拉球课程（跟踪）」）
  - 月度 summary 中「不足（待跟踪）」整段列表

### 跟踪项生命周期（示例）

**开放**（多条日志重复出现）：

```markdown
- 养成发球前思考的习惯：落点、旋转、还原、衔接（跟踪）
```

**有进步**（仍跟踪，今日写进展）：

```markdown
**收获**
- 发球前能想好落点、旋转，还原与衔接还需加强

**改善计划**
- 养成发球前思考的习惯：落点、旋转、还原、衔接（跟踪）
```

**已解决**（全文件去掉 `（跟踪）`）：

```markdown
**收获**
- 发球前四项（落点、旋转、还原、衔接）基本能想全再出手
```

历史行中的 `（跟踪）` 也应去掉，避免 Questions 重复询问。

## 月度 summary

- 位于 `## N月` 下、`### 日期` 条目之上
- 用 Obsidian callout：`> [!summary] N月总结`
- 仅在用户要求写月总结时新增/更新；日常单场复盘不改

## 时长写法

- 整数：`1h`、`2h`
- 小数：`1.5h`、`0.75h`
- 标题：`### 2026.03.05 · 0.75h`

## 对手与场景

- 公司实战常见对手：周传钦、郑锦鹏、李立杰 等（见文末表格）
- 练球：陪练、VR、球馆实战
- 内容行示例：`公司实战（周传钦、郑锦鹏）`、`陪小七练球`

## Questions 与写回联动示例

用户选择「步伐意识 → improved」后，今日条目可写：

```markdown
**收获**
- 练球时有意识小碎步调整，比上周脚更勤，但相持段仍会站死

**改善计划**
- 相持段持续小碎步微调，先动脚踩点再击球（跟踪）
```

用户选择「总结正手拉球课程 → resolved」后：

- 删除全文件 `总结正手拉球课程（跟踪）` 中的 `（跟踪）`
- 今日可写：`**收获**` `- 整理了正手拉球要点，击球点不宜抢`

## 下次重点关注（回复用户，不写日志）

复盘结束后另起小节输出 1～3 条，示例：

```markdown
### 下次重点关注

1. **反手防守别往上撅**：架住板型，用重心向前迎半出台
2. **发反手长果断快撕**：还原偏中间，除正手大角外一律反手接
3. **关键分走稳套路**：先想自己能稳得分的 1～2 套，不临时变招
```

选材：本场不足 > 仍跟踪项 > 近期重复问题；须具体、可当场执行。
