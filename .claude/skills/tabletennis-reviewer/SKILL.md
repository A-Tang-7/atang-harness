# tabletennis-reviewer

## 复盘日志

- 目标文件**以用户指定为准**（@ 引用、路径或当前打开文件）；未指定时只问一句要写入哪个文件
- 格式细节见 [references/log-format.md](references/log-format.md)

## 执行顺序（不可跳步）

1. **读日志**：读取目标文件全文；若无文件，按 references 模板新建
2. **提取跟踪项**：优先从「问题跟踪总表」（`## 问题跟踪总表`）读取状态为「跟踪中」的项；若总表不存在，再回退到扫描所有含 `（跟踪）` 的行（含月度 summary 里的待跟踪列表）。去重、归一化后得到「开放跟踪项」列表
3. **Questions 跟进（有开放项时必做）**：用 `AskQuestion` 询问；无开放项则跳过
4. **收本场输入**：用户口述/碎片信息均可；缺字段时只问关键缺失项（时长、内容），**日期必须用工具确认**（见下方日期确认规则），勿冗长问卷
5. **整理**：分点归纳，去口语重复，保留技术细节与对手名
6. **写回日志**：追加当日条目 + 按用户选择刷新全文件跟踪标记 + 同步刷新「问题跟踪总表」+ 必要时更新头部统计
7. **下次重点关注**：结合本场不足、仍开放的跟踪项及近期重复问题，给出 **3 条以内**可执行建议（见下文规则）
8. **交付摘要**：向用户说明本次写入位置、跟踪项变化、仍开放项数量，并附下次重点关注

## 日期确认规则（硬要求）

写回前**必须**执行以下日期确认，禁止凭印象或推理：

1. **当日打球（用户说"今天"）**：用 `Get-Date -Format 'yyyy.MM.dd'` 确认今日日期，再写入 `### YYYY.MM.DD`
2. **补记（用户说"补记某日"）**：以用户指定日期为准，无需工具确认
3. **多场次跨日期**：用户同时提到前几日的场次和今日场次时，每场的日期独立确认——今日场次走规则 1，历史场次走规则 2
4. **发现可疑**：若用户说"今天"但提到的对手/事件明显与已有记录冲突（如某对手已离职），提问确认后再定日期

## 跟踪项提取规则

- **首选来源**：读取 `## 问题跟踪总表` 表格中「状态」为「跟踪中」的行，提取「问题描述」列作为开放跟踪项列表
- **回退来源**：若总表不存在，则扫描所有含 `（跟踪）` 或 `(跟踪)` 的行（含月度 summary 里的待跟踪列表）
- 归一化键：去掉 `（跟踪）`、首尾空白、行首 `-`/`数字.` 后比较；合并同义表述（如「盯球意识」与「练习眼睛盯球 → 脚下移动 → 手上击球的链路」若明显同一主题，合并为一条展示，Questions 里用用户最熟悉的说法）
- 优先级（Questions 顺序）：
  1. 最近 30 天内仍出现在日志中的项
  2. 在多条日记录/月度 summary 中重复出现的项
  3. 其余开放项
- 对全部开放跟踪项逐条提问；开放项较多时可分多次 `AskQuestion` 调用，**单次会话内问完**，不留到下次

## Questions 窗口（必须用 AskQuestion）

对每条开放跟踪项发起一题，`prompt` 用简短问题句，例如：

> 「步伐意识：小碎步先踩点再击球」——本场有练/有验证吗？

**选项（每题固定提供，allow_multiple: false）**：

| 选项 id | 标签 | 含义 |
|---------|------|------|
| `resolved` | 已解决，移除跟踪 | 明显改善或已养成习惯；全文件去掉该项 `（跟踪）` |
| `improved` | 有进步，继续跟踪 | 保留 `（跟踪）`；今日条目「收获」或「改善计划」写一句进展 |
| `same` | 仍老样子，继续跟踪 | 保留 `（跟踪）`；今日「不足」或「改善计划」可写一句 |
| `skip` | 本场没练到 | 保留 `（跟踪）`；今日不写重复内容，除非用户口述有相关体会 |
| `drop` | 不再关注 | 全文件去掉该项 `（跟踪）`（与 resolved 类似，但语义为主动放弃） |

**禁止**：用聊天里连续文字代替 Questions；有开放项时必须走 `AskQuestion`。

## 本场输入 → 结构化整理

从用户输入提取并分点写入（无内容则省略该节）：

| 字段 | 写入位置 | 整理要求 |
|------|----------|----------|
| 日期 | `### YYYY.MM.DD` | 必须遵守「日期确认规则」确认后再写入；用户说"今天"则用 `Get-Date` 确认 |
| 时长 | 标题 `· Xh` | 支持 `1h` / `1.5h` / `0.75h`；不写则标题只留日期 |
| 内容 | `**内容：**` | 练球类型 + 对手/场景，一行 |
| 收获 | `**收获**` | 正面变化、有效套路、器材/手感体会 |
| 不足 | `**不足**` | 本场暴露的问题；可与跟踪项交叉引用 |
| 体会 | `**体会**` | 偏反思、心态、战术思考（与「不足」二选一或并存） |
| 改善计划 | `**改善计划**` | 可执行动作；需持续跟进的末尾加 `（跟踪）` |

整理原则：

- 一条一点，动词开头或结论开头；合并同义句
- 保留对手名、技术名词、量化（如「30 个连续」）
- 用户明确说「以后要盯这个问题」→ 对应计划行加 `（跟踪）`
- 不要编造用户未说的内容

## 写回日志

### 日条目模板

插入到对应月份区块内，**按日期升序**（同月 `## N月` 下）：

```markdown
### 2026.05.26 · 1.5h

**内容：** 公司实战（周传钦）

**收获**
- …

**不足**
- …

**改善计划**
- …（跟踪）
```

- 当月标题不存在则新建 `## N月`（中文数字月份，与现有文件一致）
- 若该日期已有条目：在条目末尾追加段落，或合并同类 bullet（勿整段覆盖，除非用户要求重写）

### 刷新跟踪标记

根据 Questions 结果**全文件**处理匹配行：

- `resolved` / `drop`：删除该行末尾 `（跟踪）`；若整行仅为跟踪占位且无实质内容，可删行（先确认）
- `improved` / `same`：保留 `（跟踪）`；在**今日**条目补充一句进展或仍存在问题
- `skip`：不改历史行；今日不强行重复

**同步刷新「问题跟踪总表」（关键步骤，禁止遗漏）**：

- 若日志中存在 `## 问题跟踪总表`，Questions 结束后必须同步更新总表对应行：
  - `resolved` / `drop`：状态改为「已解决」或「已放弃」，更新「最近更新」为今日日期，「备注」写用户反馈摘要
  - `improved` / `same` / `skip`：状态保持「跟踪中」，更新「最近更新」为今日日期，「备注」写本场进展或状态
- **新增跟踪项必须补入总表（硬要求）**：
  - 当日「改善计划」中用户明确要求新增跟踪的问题，或用户口述中首次出现、需跨场次持续跟进的问题，**必须先补入总表**，再写入当日条目
  - 补入格式：状态「跟踪中」、首次出现=今日日期、最近更新=今日日期、备注写来源（如「本场新发现」「用户明确要求跟踪」）
  - **禁止只写当日条目而漏写总表**；若总表不存在，则当日条目中的 `（跟踪）` 标记仍保留，但需在交付摘要中提示用户建立总表
- **排序与删除线**：总表必须始终按「首次出现」日期升序排列；状态为「已解决」或「已放弃」的行，问题描述列加删除线 `~~描述~~`，并移到表格末尾

**写回前自检清单（必须执行）**：
1. 当日条目中所有含 `（跟踪）` 的行，是否都能在总表中找到对应项？
2. 若当日新增了 `（跟踪）` 项，总表是否已补入？
3. 总表「最近更新」列是否为今日日期？
4. 若以上任一答案为「否」，禁止进入「交付摘要」步骤，必须先补齐。

同步规则：

- 月度 `> [!summary]` 里「不足（待跟踪）」列表与正文 `（跟踪）` 保持一致：已 resolved 的从 summary 删除或改措辞
- 同一主题在 summary 与子条目重复时，resolved 后两处都更新

### 头部统计（每次写回时更新）

更新 frontmatter 下方统计行（若存在）：

```markdown
> 统计截至 YYYY.MM.DD，累计 **N 次**打球，约 **X 小时**。
```

- `N`：`### YYYY.MM.DD` 条目数（去重日期）
- `X`：各条目标题中 `· Xh` 时长之和（解析失败则跳过该条）
- 「统计截至」用本次复盘日期或日志中最后一场日期

### 编码

- 读写的 `.md` 与源文件编码一致；新建默认 UTF-8（无 BOM）

## 下次重点关注（必选，≤3 条）

写回日志并完成整理后，**必须**在回复中单独给出「下次打球重点关注」，不写入日志文件（除非用户明确要求追加到「改善计划」）。

### 选材范围

按优先级从复盘记录中抽取：

1. **本场「不足」**：今日刚暴露、尚未解决的具体问题
2. **仍开放的 `（跟踪）` 项**：Questions 结果为 `same` / `improved` / `skip` 的项
3. **近期重复主题**：近 5～10 场日志中多次出现的同类不足（如步伐、盯球、盲目加力）

合并同主题后取最关键的 **1～3 条**；不足 3 条时不凑数。

### 每条格式

每条一行标题 + 一句可执行提醒，场上能立刻想起来：

```markdown
### 下次重点关注

1. **发球前想全四项**：落点、旋转、还原、衔接——发之前停半拍
2. **相持先动脚**：算到落点就先小碎步踩位，再出手
3. **机会球控力**：能上台的加速，不盲目加力打丢
```

要求：

- 来自日志中的真实不足，不泛泛而谈（避免「多练习」「提高意识」等空话）
- 偏**下场可验证**的动作/意识/战术，而非长期装备或课程规划
- 若下场对手已知（本场内容里写过），可点一句针对性场景（如「对周传钦少劈长、优先拧中间」）
- 与今日「改善计划」重复时可合并表述，勿机械复制日志原文

## 交付摘要（回复用户）

用短列表说明：

1. 已写入/更新的日期与路径
2. 本场 3～5 条要点（收获 + 最优先改善）
3. 跟踪项：本次 resolved / 仍开放数量
4. **下次重点关注**（1～3 条，单独小节，格式见上）

## 边界

- 不自动写月度 summary（用户明确要求「写 X 月总结」时再写）
- 不修改「常见对手分析」表，除非用户当场提到对手新策略且要求更新
- 写回前确认用户指定的复盘日志路径，避免误写副本
