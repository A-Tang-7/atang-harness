# 视频字幕汇总 Skill

## 功能说明

读取 `raw/` 目录下所有 B 站视频字幕文件，对每个视频内容进行完整总结，将所有视频的总结合并生成一份汇总报告，输出到 `output/` 目录。

## 触发方式

- `/video-summary` — 处理 `raw/` 中所有视频
- `/video-summary <文件名>` — 只处理指定的视频文件

## 执行流程

### 1. 扫描文件

列出 `raw/` 目录下所有 `.md` 文件。如果用户指定了文件名，只处理该文件。

### 2. 解析每个文件

每个字幕文件的前 3 行为元数据：

```
视频标题：<标题>
视频作者：<作者>
发布日期：<YYYY-MM-DD>
```

之后是 `---\n字幕正文\n---` 分隔符，分隔符之后为完整的字幕正文。

提取以下字段：
- **标题**：视频标题
- **作者**：视频作者
- **日期**：发布日期
- **字幕正文**：分隔符之后的所有文本

### 3. 一句话总结

在完整阅读每个视频的字幕正文后，先用 **一句话**（不超过 80 字）概括视频的核心观点或核心价值。要求：
- 高度凝练，直击要害
- 让人一眼就能判断这个视频是否值得细读
- 放在内容总结的最前面

### 4. 内容总结（关键规则）

对每个视频的字幕正文进行深度总结，**必须遵守以下规则**：

1. **不删减**：保留视频中提到的每一个观点、论点、案例、数据和结论，不得遗漏任何实质性内容
2. **分点提炼**：将视频内容按照逻辑结构进行分点整理，使用层级标题和小点
3. **去口语化**：将口语表达转换为书面语言，修正明显的语音识别错误，但保持原意不变
4. **保留原话**：对于关键的金句、核心观点，可以保留原文引用
5. **结构清晰**：按主题/逻辑分段，使用 markdown 的 `###`、`-`、数字列表等层级结构
6. **不添加**：不要添加视频中没有的观点，不要做主观评价

### 4. 生成报告

将所有视频的总结合并写入 `output/` 目录下的**单个 `.md` 文件**中。

文件命名格式：`视频总结报告-YYYY-MM-DD.md`（使用当前日期）

### 报告结构

```markdown
# B站视频内容总结报告

生成日期：YYYY-MM-DD
视频总数：N 个

---

## 1. [视频标题]

**作者**：xxx | **发布日期**：YYYY-MM-DD

> **一句话**：用不超过 80 字概括本视频的核心观点

### 内容总结

（分点提炼的完整内容...）

---

## 2. [视频标题]

**作者**：xxx | **发布日期**：YYYY-MM-DD

> **一句话**：用不超过 80 字概括本视频的核心观点

### 内容总结

（分点提炼的完整内容...）

---
```

### 5. 完成确认

报告生成后，告知用户：
- 处理了多少个视频
- 输出文件路径
- 报告总字数/行数

## 注意事项

- 每个视频的字幕正文可能很长（数千至数万字），务必完整阅读全部内容后再总结
- 字幕来自语音识别，可能包含口语填充词（"是吧"、"那个"、"然后"）、重复语句、识别错误，总结时需要去噪但保留所有观点
- 如果字幕中出现明显的专有名词识别错误（如"聚村智能"应为"具身智能"、"A i m"应为"AI"），应在总结中修正
- 输出文件统一使用 UTF-8 编码
